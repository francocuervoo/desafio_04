El formulario es solo para cargar productos a la lista
No es necesario que tenga la opción de modificación de productos ya existentes

Cuando dice "Implementar la API en una clase separada" lo que quiere decir es que se debe utilizar la clase "Contenedor" del Desafío 3 y que ésta sea exportada del backend donde está el código de Express utilizado

Pero en este caso la lista no debe quedar guardada en un archivo txt sino en un archivo json

No es necesario usar Multer. Para el formulario de carga de datos pueden usar una URL en vez de la imagen misma. Se puede usar www.imgur.com

Para el caso de que un producto no exista, se devolverá el objeto:
{ error : 'producto no encontrado' }

Implementar la API en una clase separada, utilizando un array como soporte de persistencia en memoria.

Incorporar el Router de express en la url base '/api/productos' y configurar todas las subrutas en base a este.

Crear un espacio público de servidor que contenga un documento index.html con un formulario de ingreso de productos con los datos apropiados.

El servidor debe estar basado en express y debe implementar los mensajes de conexión al puerto 8080 y en caso de error, representar la descripción del mismo.

Las respuestas del servidor serán en formato JSON. La funcionalidad será probada a través de Postman y del formulario de ingreso.
